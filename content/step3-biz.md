# Phase 1: 비즈니스 모델 이해

## EXPLAIN

### 콜라보는 어떻게 돈을 벌까?

콜라보는 **B2B SaaS 구독 모델**입니다. 기업 고객이 워크스페이스를 만들고, 팀원을 초대해서 사용하며, 사용 규모에 따라 요금을 지불합니다.

### 매출 구조

| 플랜 | 가격 | 대상 | 특징 |
|------|------|------|------|
| **Free** | 무료 | 소규모 팀 (10명 이하) | 기본 화이트보드 + 프로젝트보드, 제한된 저장공간 |
| **Pro** | $15/user/월 | 성장하는 팀 | 무제한 멤버, 고급 연동, 분석 대시보드 |
| **Enterprise** | 맞춤 견적 | 100명+ 대기업 | SSO, 전담 CSM, SLA 보장, 커스텀 연동, 감사 로그 |

> 💡 **매출 비중**: Enterprise 30% + Pro 60% + 부가서비스 10%
> Enterprise 고객 수는 적지만, 객단가가 높아 전체 매출의 30%를 차지합니다.

### 핵심 지표

| 지표 | 의미 | 현재 수준 |
|------|------|-----------|
| **MRR** (월간 반복 매출) | 매달 안정적으로 들어오는 구독 수익 | 약 8억원 |
| **ARR** (연간 반복 매출) | MRR × 12, 투자자가 가장 중요시하는 지표 | 약 96억원 |
| **NRR** (순매출유지율) | 기존 고객의 확장/축소/이탈을 반영한 수익 유지율 | 115% (확장이 이탈보다 큼) |
| **Churn Rate** (이탈률) | 월간 유료 고객 이탈 비율 | 2.5% |
| **DAU** (일간 활성 사용자) | 매일 콜라보를 사용하는 유저 수 | 약 3만명 |

### 경쟁 포지셔닝

| 서비스 | 유형 | 특징 | 콜라보와의 차이 |
|--------|------|------|----------------|
| **Notion** | 올인원 워크스페이스 | 문서 + DB + 위키 | 실시간 협업/화이트보드 약함 |
| **Miro** | 화이트보드 특화 | 비주얼 협업에 강점 | 프로젝트 관리 기능 없음 |
| **Monday.com** | 프로젝트 관리 | 워크플로우 자동화 강점 | 화이트보드/실시간 협업 없음 |
| **콜라보** | **화이트보드 + PM 통합** | 아이디어 → 실행을 하나의 도구에서 | 두 영역을 실시간으로 연결 |

> 콜라보의 핵심 차별점: **"아이디어가 바로 태스크가 된다"** — 화이트보드에서 브레인스토밍한 내용이 클릭 한 번으로 프로젝트 보드의 태스크로 변환됩니다.

### 고객 여정 요약

```
무료 가입 → 팀 초대 → 활성 사용 → Pro 전환 → 사용량 증가 → Enterprise 문의 → 전담 CSM 배정
```

# Phase 2: 시나리오 퀘스트 — BM 실전 적용 + 팀 연결

## EXPLAIN

### 왜 시나리오 퀘스트를 하나요?

Phase 1에서 콜라보의 BM을 숫자와 구조로 배웠어요.
이번에는 **실제 업무 상황**에서 BM이 어떻게 작동하는지 체험해볼 거예요.

시나리오 퀘스트는 팀스페이스에서 실제로 일어나는 상황을 바탕으로 만든 미니 케이스 스터디예요.
정답이 있는 건 아니에요 — 여러분의 관점으로 생각하고, 어떤 판단을 내릴 수 있는지 탐색하는 과정입니다.

### 고객 여정 상세

시나리오를 풀기 전에, 콜라보의 고객 여정을 좀 더 자세히 알아볼게요:

```
1. 발견 → 검색/추천으로 콜라보를 알게 됨
2. 무료 가입 → 워크스페이스 생성 + 팀원 3~5명 초대
3. 활성화 → 화이트보드 or 프로젝트보드를 주 3회 이상 사용
4. Pro 전환 → 10명 초과 또는 고급 기능 필요 시 유료 전환
5. 확장 → 사내 다른 팀으로 확산, 멤버 수 증가
6. Enterprise → 100명+ 사용, 보안/관리 요구 → Enterprise 문의
7. 온보딩 세션 → Onboarding셀이 전담하여 도입 지원
8. 갱신 → 연간 계약 갱신, NRR 115%+ 유지
```

### Support셀의 역할

고객 여정 전체에서 **Support셀**(리드: Mia)이 고객 접점을 관리합니다:
- **일반 문의**: 인앱 채팅 + 이메일 → 24시간 내 응답
- **Enterprise 장애**: `#cs-enterprise-support` Slack 채널 → SLA 기준 1시간 내 초동 대응
- **에스컬레이션**: 기술 이슈 → Engineering팀 Backend셀(Kai) 또는 Frontend셀(Noah) 태그
- **VOC 수집**: 고객 피드백을 Product팀에 정기 리포트
