# Step 3: Biz Onboarding — 폴더 컨텍스트

## Phase 구조 (4개)

이 Step은 다른 Step보다 Phase가 많다 (Phase 0~3).
각 Phase의 references 파일을 순서대로 읽되, STOP PROTOCOL을 반드시 지킨다.

## 팀별 분기 주의사항

- Phase 2: 신규입사자의 **소속 팀 업무 → 타 팀 연계** 시나리오 2개 제공
- Phase 3: 신규입사자의 **소속 팀** 심화 BM 연결

`progress.json`에서 팀 정보를 확인하고, 없으면 AskUserQuestion으로 묻는다.

## Notion 참조

`config/notion-ids.json`의 `step3_biz` ID 사용.
Phase 1의 BM 정보는 Notion에서 최신 데이터를 가져와 보충한다.

## 경쟁 포지셔닝 핵심

콜라보는 **화이트보드 + 프로젝트 관리 통합** SaaS.
- vs Notion (올인원 but 실시간 협업/화이트보드 약함)
- vs Miro (화이트보드 강점 but 프로젝트 관리 없음)
- vs Monday.com (PM tool but 화이트보드/실시간 협업 없음)
