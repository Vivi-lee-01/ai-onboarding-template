# Onboarding셀 시나리오 (→ Product / Support 연계)

> 시나리오 A — Onboarding → Product
>
> Enterprise 고객 "Acme Corp"의 온보딩 후 활성 사용률이 급락했습니다.

## EXPLAIN

### 상황

Onboarding셀(리드: Ava)이 3주 전에 Acme Corp(1,200명)의 온보딩 세션을 완료했습니다. 초기 활성화 지표는 양호했지만, 2주차부터 DAU가 급락하여 현재 도입 전 대비 15%만 활성 사용 중입니다.

원인 후보:
- 온보딩 세션이 관리자 중심이라 일반 멤버 교육이 부족했을 수 있음
- 기존 도구(Miro+Asana)에서의 마이그레이션이 완전하지 않아 이중 사용 중
- 부서별 사용 편차가 큼 (마케팅팀은 활발, 엔지니어링팀은 미사용)

## EXECUTE

> **에이전트 지시**:
> 1. 위 상황을 설명한 후, 아래 질문을 AskUserQuestion으로 묻는다
> 2. 온보딩 프로세스 개선 관점에서 토론을 유도한다

"여러분이 Onboarding셀 담당이라면, 활성 사용률을 회복시키기 위해 어떤 조치를 취하시겠어요?"

## CHECK

- 다양한 관점 제시:
  - **데이터 분석**: 부서별/기능별 사용 패턴 확인 → 어디서 이탈이 발생하는지 특정
  - **추가 세션**: 일반 멤버 대상 "사용법 워크숍" 추가 진행
  - **마이그레이션 완료**: 기존 Miro/Asana 데이터를 완전히 옮기고, 이전 도구 접근을 제한하도록 관리자에게 제안
  - **Product 피드백**: 엔지니어링팀이 안 쓰는 이유 → 제품 개선 요청 (예: GitHub 연동 강화)
- "온보딩의 성공 기준은 '세션 완료'가 아니라 '자력으로 활발히 사용하는 상태'예요"

---

> 시나리오 B — Onboarding → Support
>
> 금융권 Enterprise 고객 킥오프에서 SSO 연동 장애가 발생했습니다.

## EXPLAIN

### 상황

금융권 대기업 "FinSecure"(2,000명)의 킥오프 온보딩 세션 당일, SSO(SAML) 연동이 작동하지 않습니다. 고객사 보안팀이 방화벽에서 콜라보 인증 서버 IP를 화이트리스트에 추가하지 않은 것으로 추정됩니다.

킥오프에 참석한 고객사 임원 3명이 "이런 기본적인 것도 안 되나"며 불만을 표시하고 있습니다.

## EXECUTE

"킥오프 현장에서 이 상황이 벌어졌다면, 여러분은 어떻게 대응하시겠어요?"

## CHECK

- 다양한 관점 제시:
  - **즉시 대응**: 고객사 보안팀에 IP 화이트리스트 요청 + 임시 접속 방법(비SSO 초대 링크) 제공
  - **임원 관리**: "사전 체크리스트에서 누락된 점 사과드립니다" + 5분 내 해결 계획 공유
  - **근본 해결**: 킥오프 전 체크리스트에 "방화벽 화이트리스트 확인" 항목 추가
  - **Support 연계**: Backend셀(Kai)에 긴급 지원 요청
- "현장 장애 대응에서 가장 중요한 건 '기술적 해결'보다 '고객의 불안감 해소'가 먼저예요"
