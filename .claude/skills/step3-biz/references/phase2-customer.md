# Phase 2: 시나리오 퀘스트 — BM 실전 적용 + 팀 연결

## EXPLAIN

### 왜 시나리오 퀘스트를 하나요?

Phase 1에서 콜라보의 BM을 숫자와 구조로 배웠어요.
이번에는 **실제 업무 상황**에서 BM이 어떻게 작동하는지 체험해볼 거예요.

시나리오 퀘스트는 팀스페이스에서 실제로 일어나는 상황을 바탕으로 만든 미니 케이스 스터디예요.
정답이 있는 건 아니에요 — 여러분의 관점으로 생각하고, 어떤 판단을 내릴 수 있는지 탐색하는 과정입니다.

### 고객 여정 상세

시나리오를 풀기 전에, 콜라보의 고객 여정을 좀 더 자세히 알아볼게요:

```
1. 발견 → 검색/추천으로 콜라보를 알게 됨
2. 무료 가입 → 워크스페이스 생성 + 팀원 3~5명 초대
3. 활성화 → 화이트보드 or 프로젝트보드를 주 3회 이상 사용
4. Pro 전환 → 10명 초과 또는 고급 기능 필요 시 유료 전환
5. 확장 → 사내 다른 팀으로 확산, 멤버 수 증가
6. Enterprise → 100명+ 사용, 보안/관리 요구 → Enterprise 문의
7. 온보딩 세션 → Onboarding셀이 전담하여 도입 지원
8. 갱신 → 연간 계약 갱신, NRR 115%+ 유지
```

### Support셀의 역할

고객 여정 전체에서 **Support셀**(리드: Mia)이 고객 접점을 관리합니다:
- **일반 문의**: 인앱 채팅 + 이메일 → 24시간 내 응답
- **Enterprise 장애**: `#cs-enterprise-support` Slack 채널 → SLA 기준 1시간 내 초동 대응
- **에스컬레이션**: 기술 이슈 → Engineering팀 Backend셀(Kai) 또는 Frontend셀(Noah) 태그
- **VOC 수집**: 고객 피드백을 Product팀에 정기 리포트

## EXECUTE

> **에이전트 지시**:
> 1. 위 고객 여정을 설명한다
> 2. 신규입사자의 소속 팀에 맞는 시나리오 파일 2개를 선택한다:
>    - 1번째: **소속 팀 업무** 시나리오 (자기 팀 관점)
>    - 2번째: **타 팀 연계** 시나리오 (협업 관점)
>    - 시나리오 파일: `phase2-scenarios-{팀}.md`
>    - 팀 매핑: product, sales, dev, cx, mgmt, rocket(=onboarding)
> 3. 각 시나리오를 STOP PROTOCOL에 따라 진행 (시나리오 1개 = 1 Phase A/B 사이클)
> 4. 시나리오에 정답은 없다 — 다양한 관점을 제시하고 토론을 유도

이제 시나리오 퀘스트를 시작할게요! 여러분의 팀과 관련된 실전 상황이 나올 거예요.
