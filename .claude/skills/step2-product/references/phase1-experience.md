# Phase 1: 워크스페이스 오너 + 멤버 체험

## EXPLAIN

### 개밥먹기(Dogfooding)란?

"개밥먹기"는 자기가 만든 제품을 직접 써보는 것입니다.
우리 제품을 직접 써봐야 고객에게 제대로 설명할 수 있고, 진짜 개선점을 발견할 수 있어요.

콜라보에는 두 종류의 핵심 사용자가 있습니다:
- **워크스페이스 오너(Admin/Owner)**: 팀 워크스페이스를 만들고 멤버·권한·보드 구조를 설계하는 관리자
- **협업 멤버(Member)**: 화이트보드와 프로젝트 보드에서 아이디어를 만들고 태스크를 실행하는 팀 구성원

직무와 관계없이, 모든 구성원이 **양쪽 관점**을 이해해야 합니다.

### 체험 + 기록을 동시에

이번 Phase에서는 체험과 기록을 **동시에** 진행합니다.
Notion에 **개밥먹기 노션 문서**를 열어두고, 체험하면서 느낀 점을 바로바로 적어주세요.

나중에 한꺼번에 쓰면 기억이 흐려지지만, **체험하면서 바로 적으면** 생생한 인사이트가 남아요!

기록할 때 아래 관점을 참고하세요:
- **막힌 부분**: 어디서 헤맸는지, 뭐가 헷갈렸는지
- **원활한 부분**: 어디가 직관적이고 쉬웠는지
- **설계 의도 추정**: "왜 이렇게 만들었을까?" 나름의 추측
- **워크스페이스 오너 vs 협업 멤버 비교**: 양쪽 경험의 차이점

## EXECUTE

> 이번 Phase에서는 🌐 **Notion 브라우저**(체험 가이드 + 개밥먹기 기록)와 🌐 **콜라보/Collabo Academy 사이트**(체험)를 사용합니다. 체험이 끝나면 Claude Code로 돌아와주세요.

> **에이전트 지시** (신규입사자에게 보여주지 않는 내부 메모):
>
> **⚠️ 체험 가이드와 개밥먹기 문서는 서로 다른 페이지다. 반드시 구분하여 안내한다.**
>
> **1단계: 체험 가이드 열기** (공용 문서 — 모든 대상자가 같은 페이지)
> - `config/notion-ids.json`의 `step2_product` ID로 URL을 조립해 브라우저로 연다 (fetch 불필요)
> - 직접 가이드 내용을 요약하지 않는다 — **Notion 페이지 링크로 안내**하여 원본을 직접 읽게 한다
> - 보충 설명이 필요하면 `content/step2-product-guide.md`(로컬 사본)를 참조한다
>
> **2단계: 개밥먹기 문서 열기** (개인 문서 — 대상자별 다른 페이지)
> - `progress.json`의 `dogfooding_page_id`를 확인한다
> - ID가 있으면: 해당 페이지 URL을 브라우저로 열어준다
> - ID가 null이면: `notion-search`로 "{닉네임} 개밥먹기"로 재검색 → 찾으면 progress.json 업데이트
> - 그래도 못 찾으면: "HR(Dana)에게 개밥먹기 노션 문서를 요청해주세요" 안내
>
> **금지 사항:**
> - fetch 실패 시 fallback: "Notion에서 'Product Onboarding'을 검색하면 체험 가이드를 찾을 수 있어요"
> - ⚠️ 콜라보를 강의 마켓플레이스/노코드 사이트 빌더처럼 설명하지 않는다. "강의 신청", "수강", "사이트 만들기" 식의 안내 금지.
>   콜라보는 팀이 화이트보드에서 만든 아이디어를 프로젝트 보드의 실행 태스크로 연결하는 실시간 협업 SaaS다.

### 체험 순서

**두 개 탭**을 Notion에서 열어두세요:
- **체험 가이드** — 체험 순서를 따라가는 용도
- **개밥먹기 노션 문서** — 느낀 점을 바로 적는 용도

**미션 1: 워크스페이스 오너 체험**
1. 회사 계정이 아닌 **개인 계정**으로 콜라보에 회원가입
2. 워크스페이스 오너로서 워크스페이스를 만들고 멤버 초대·권한·보드 구조를 설정해보세요 (Notion 체험 가이드 참고)
3. 체험하면서 막힌 점/원활한 점을 **개밥먹기 노션 문서에 바로 기록**

**미션 2: 협업 멤버 체험**
1. 샘플 워크스페이스에서 멤버 관점으로 보드에 참여해보세요
2. 화이트보드 아이디어 작성 → 태스크 전환 → 프로젝트 보드 상태 변경 Flow를 체험해보세요 (Notion 체험 가이드 참고)
3. **같은 개밥먹기 노션 문서**에 멤버 체험 내용도 이어서 기록
   - "워크스페이스 오너로는 이랬는데 멤버로는 이렇다" 식의 비교 메모가 특히 좋아요!
