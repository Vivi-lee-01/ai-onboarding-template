# Phase 2: Enterprise 온보딩 세션 참관 리포트

## EXPLAIN

### 참관 경험을 리포트로 남기는 이유

Enterprise 온보딩 세션을 참관하면서 느낀 점, 관찰한 내용, 개선 아이디어를 구조화해서 남기면:

1. **Onboarding셀에 실전 피드백**을 전달할 수 있어요 (프로세스 개선)
2. **Product팀에 고객 관점 인사이트**를 공유할 수 있어요 (제품 개선)
3. **본인의 학습을 정리**하는 효과가 있어요 (온보딩 마무리)

### 리포트 구조

아래 구조로 작성합니다:

```markdown
# Enterprise 온보딩 세션 참관 리포트 — {닉네임}

## 기본 정보
- 참관 일시: {날짜, 시간}
- 고객사: {고객사명}
- 세션 유형: {초기 셋업 / 팀원 교육 / 연동 설정 / 커스텀 템플릿}
- 참석자: Onboarding셀 담당자, 고객사 참석자 수

## 1. 온보딩 프로세스 관찰
- 세션이 어떤 순서로 진행됐는지
- 특히 효과적이었던 부분
- 개선되면 좋겠다고 느낀 부분

## 2. 고객 반응
- 고객이 가장 관심을 보인 기능
- 고객이 어려워하거나 질문이 많았던 부분
- 기대와 현실의 갭이 있었던 부분

## 3. 실시간 대응
- 세션 중 예상 못한 상황이 있었는지
- Onboarding셀이 어떻게 대응했는지

## 4. 제품 피드백
- 고객 관점에서 느낀 콜라보의 강점
- 개선이 필요하다고 느낀 기능/UX

## 5. 서비스 UX
- 초기 사용자 관점에서 직관적이었던 부분
- 혼란스러웠거나 설명이 필요했던 부분

## 6. Onboarding셀 / Product팀에 제안
- 온보딩 프로세스 개선 아이디어
- 제품 개선 제안
```

---

## EXECUTE

> **에이전트 지시** (신규입사자에게 보여주지 않는 내부 메모):
>
> **리포트 작성 플로우**:
> 1. 신규입사자에게 위 리포트 구조를 보여주고, 각 섹션을 대화형으로 채워간다
> 2. 한 섹션씩 AskUserQuestion으로 "이 부분은 어떠셨어요?"라고 물어보며 진행
> 3. 충분히 수집되면 리포트를 통합 정리
> 4. 리포트를 `outputs/{닉네임}-{입사일}/step6-session-report.md`에 저장
> 5. Notion에도 저장 시도:
>    - progress.json에서 `notion_kanban_ds` 읽기
>    - 칸반에서 "Step.6" 또는 "참관" 카드를 찾아 하위 페이지로 리포트 생성
>    - Notion 저장 실패 시: 로컬 파일만으로 OK
> 6. progress.json의 `step6_status`를 `"completed"`로 설정

리포트를 함께 작성해볼게요! 세션에서 관찰한 내용을 하나씩 정리해봐요.
