# Phase 1: Enterprise 온보딩 세션 참관 준비

## EXPLAIN

### Enterprise 온보딩 세션 참관, 왜 해야 하나요?

팀스페이스 구성원이라면 **Enterprise 고객의 온보딩 세션을 꼭 한번은 참관**해야 합니다.

콜라보 매출의 **30%를 차지하는 Enterprise 고객**의 온보딩 세션을 직접 관찰하면서:

- Onboarding셀이 어떻게 고객사에 콜라보를 안착시키는지
- 고객사 관리자와 멤버가 어떤 질문을 하고 어떤 어려움을 겪는지
- 워크스페이스 초기 설정, SSO 연동, 커스텀 템플릿 배포가 어떻게 진행되는지

를 직접 눈으로 확인하는 과정입니다.

Step 2에서 제품을 직접 체험하고, Step 3에서 BM/고객 여정을 학습했는데,
이제 **실제 고객 현장에서 그것이 어떻게 동작하는지** 확인하는 겁니다.

### 참관 일정 안내

Enterprise 온보딩 세션은 보통 **오후 2~4시**에 진행됩니다 (고객사 시간대에 맞춤).

- **일반 세션**: 업무 시간 내이므로 별도 근무 조정 불필요
- **해외 고객 세션**: 시차에 따라 유연근무 가능 (별도 증빙 불필요)
- **팀 내 공유 필수**: 참관 일정을 팀 채널에 미리 공유

### 참관 채널

Slack `#cs-enterprise-support` 채널에서:
- Onboarding셀이 세션 일정을 공유
- 세션 중 이슈 발생 시 실시간 대응 스레드 운영

> 3. Notion 원본 링크도 함께 공유: "더 자세한 내용은 이 Notion 문서에서 확인할 수 있어요!"
> 4. fetch 실패 시: 아래 내용만으로 충분히 안내 가능 (fallback)

### 1. 예정된 세션 확인

1. **콜라보 Admin** (admin.collabo.io)에 회사 구글 계정으로 로그인
   - 로그인이 안 되면 → BE셀 **Kai**에게 Slack으로 권한 요청
2. Enterprise 고객 관리 → 온보딩 세션 → 상태 "예정" 필터
3. 이번 주~다음 주 예정된 세션 중 참관할 세션을 선택

**추천 세션 유형:**
- **초기 셋업 세션**: 워크스페이스 구조 설계 + 관리자 교육
- **팀원 교육 세션**: 일반 멤버 대상 사용법 안내
- **연동 설정 세션**: SSO, Slack/GitHub 연동 등 기술 설정
- **커스텀 템플릿 세션**: 고객사 맞춤 템플릿 제작 + 배포

참관 대상 선정이 어려우면 Onboarding셀(Ava) 또는 HR(Dana)에게 문의하세요.

### 2. 참관 준비

1. Slack `#cs-enterprise-support` 채널에 가입 (이미 가입되어 있을 수 있음)
2. Onboarding셀(Ava)에게 Slack DM으로 참관 요청: "이번 주 {고객사} 온보딩 세션 참관하고 싶어요!"
3. 참관할 세션 일정을 **팀 채널에 공유**
4. 아래 관찰 포인트를 미리 숙지

### 3. 관찰 포인트

세션 참관 시 아래 관점으로 관찰하며 메모하세요:

1. **온보딩 프로세스**: Onboarding셀이 어떤 순서로 안내하는지
2. **고객 반응**: 관리자/멤버가 어떤 질문을 하는지, 어디서 막히는지
3. **실시간 대응**: 예상 못한 상황(연동 오류, 기능 질문 등)에 어떻게 대응하는지
4. **제품 피드백**: 고객이 기대한 기능 vs 실제 기능의 갭은 없는지
5. **서비스 UX**: 초기 사용자 관점에서 느낀 직관성, 불편한 점

### 4. 자동 리마인더 설정

에이전트가 Phase 1 CHECK 완료 시 자동으로 수행:

1. **세션 날짜를 확인**한다 (신규입사자에게 "세션이 몇 월 며칠인가요?" 질문)
2. **세션 당일 오후 5시**에 신규입사자에게 Slack 예약 메시지 발송:
   > "오늘 Enterprise 온보딩 세션 참관 수고 많았어요! :tada: 시간 될 때 `/onboarding 이어하기`로 돌아와서 참관 리포트를 작성해주세요. 리포트까지 완료하면 온보딩이 최종 마무리됩니다!"
3. **HR(Dana)에게도 DM 발송**:
   > ":eyes: *Enterprise 세션 참관 알림*\n{닉네임} ({직무}, {팀})이 {날짜} Enterprise 온보딩 세션 참관 예정입니다.\n참관 후 리포트 미작성 시 팔로업이 필요할 수 있어요."
4. **리포트 미작성 팔로업**: 세션 **3영업일 후**에도 progress.json의 step6_status가 `"phase1_done"`이면, HR(Dana)에게 추가 알림:
   > ":warning: *온보딩 팔로업 필요*\n{닉네임}이 Enterprise 세션 참관 후 3영업일이 지났으나 리포트를 아직 작성하지 않았어요."

> 리마인더 발송 도구: `slack_schedule_message` MCP 사용
> 3영업일 팔로업은 에이전트가 자동으로 보낼 수 없으므로 → HR(Dana)에게 직접 팔로업을 위임
