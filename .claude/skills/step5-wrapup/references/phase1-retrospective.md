# Phase 1: 캠프 회고 + VoC

## EXPLAIN

### 회고의 목적

온보딩 캠프 전체를 돌아보고, 앞으로의 방향을 정리하는 짧은 시간입니다.
형식보다 본인의 생각이 중요해요. 회고와 함께 온보딩 프로그램에 대한 솔직한 피드백도 들려주세요 — 다음 입사자의 온보딩을 더 좋게 만드는 데 쓰여요.

## EXECUTE

> **에이전트 지시**:
> 1. outputs/ 디렉토리의 산출물 목록을 간단히 보여준다 (파일명만, Notion 실패 시 fallback 저장분만 있을 수 있음 — 파일이 없으면 "산출물은 모두 Notion에 저장되어 있어요"로 안내)
> 2. "이 중에서 가장 기억에 남는 순간이 있나요? 그리고 온보딩에서 좋았던 점이나 아쉬웠던 점도 자유롭게 말해주세요!" 하나의 오픈 질문으로 시작한다
> 3. 신규입사자가 자유롭게 이야기하도록 한다 — 구조화된 템플릿으로 유도하지 않는다
> 4. 답변이 짧으면 자연스럽게 한두 번만 더 물어본다 (최대 2회 추가 질문)
> 5. 회고 내용은 progress.json에 기록된다 (별도 파일 저장 불필요)

온보딩에서 만든 산출물들을 한번 돌아보면서, 가장 기억에 남는 것과 좋았던 점/아쉬웠던 점을 자유롭게 이야기해주세요.
